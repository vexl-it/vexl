# AGENTS

Purpose: Shared UI package providing the Vexl design system built on Tamagui. Native-only (no web/media queries).

Stack: TypeScript, Tamagui v1, React, react-native-reanimated v4.

Structure:

- `src/config/` — Tamagui configuration (tokens, themes, fonts).
- `src/provider/` — VexlThemeProvider and theme hooks.
- `src/primitives/` — Re-exported Tamagui primitive components (Stack, Text, etc.).
- `src/assets/anonymousAvatars/` — Auto-generated anonymous avatar SVG components (31 basic + 27 golden). Each accepts `size` and `grayscale` props. Grayscale uses SVG `<Filter>`/`<FeColorMatrix type="saturate" values="0">` wrapped in `<G filter>`. Generated by `scripts/convert-avatars.mjs` from the mobile app's SVG strings. Exported as `avatarsSvg` and `avatarsGoldenGlassesAndBackgroundSvg` arrays. Files use `// prettier-ignore` on the return statement to prevent Prettier from inserting `{' '}` text nodes (which crash in react-native-svg).
- `src/components/` — Custom Vexl design-system components:
  - `Button` — Primary action button (5 variants × 3 sizes).
  - `CardButton` — Small card-level button (filled/text × contrast flag). Filled+contrast uses theme tokens; filled without uses primitives ($black100/$white100).
  - `FabButton` — Floating action button (icon + label, always yellow primary).
  - `Dot` — Small circle indicator. 2 variants: `small` (8px, default) and `number` (20px, shows a `count` label). Default background `$pinkBright100`, settable via `backgroundColor` prop. Number label uses plain RN `Text` (not `SizableText`) with font values resolved from `bodyFont` config to avoid Tamagui lineHeight misalignment inside the circle.
  - `IconButton` — Square icon button with optional badge (uses `Dot` with `size="$5"` and `$accentYellowPrimary` background).
  - `NavButton` — Navigation bar button (icon/text × highlighted/destructive/normal). Normal variant wraps in `<Theme name="light">` to stay light in dark mode.
  - `Avatar` — Rectangular avatar with border radius `$2.5` (8px). 3 sizes (small/medium/large) + customSize. Supports raster image (`source` prop, grayscale via `FilterImage` from `react-native-svg/filter-image`) or children (icons, SVG avatar components). For children, grayscale is handled by the child component itself (e.g. anonymous avatar `grayscale` prop).
  - `FilterTag` — Pill-shaped filter tag (default/selected states). No press feedback (no `pressStyle`) — behaves like `TouchableWithoutFeedback`. Optional `icon` prop (`React.ComponentType<IconProps>`) — pass the icon component reference (e.g. `icon={TagLabel}`), color and size are controlled by FilterTag internally.
  - `FilterBar` — Horizontally scrollable row of `FilterTag` items. Generic over `T` — accepts `items: ReadonlyArray<FilterBarItem<T>>` (each with `label` + `value`), `selectedValues: ReadonlySet<T>`, `onSelectedValuesChange: (values: ReadonlySet<T>) => void`. Composes `FilterTag` directly (no inline styled duplicates). Auto-scrolls to reveal toggled tags via `onLayout` measurements. Wrapped in RN `ScrollView` (horizontal). Gap between tags resolved from `$3` space token.
  - `DotTypingIndicator` — Animated typing indicator (3 bouncing dots) using react-native-reanimated.
  - `Exchange` — Currency exchange input with two fields (BTC/SATS top, Fiat bottom) and a decorative center arrows badge. Uses controlled props (not jotai atoms) — `btcValue`/`fiatValue` strings + `onBtcValueChange`/`onFiatValueChange` callbacks — so the parent can coordinate exchange-rate recalculations. BTC/SATS toggle (`onBtcUnitChange`) switches the top label and keyboard type (`number-pad` for SATS, `decimal-pad` for BTC). Fiat currency label + chevron triggers `onFiatCurrencyPress` (for external currency picker). Numbers formatted via `Intl.NumberFormat(locale)` when input is blurred; raw string shown when focused. `btcEditable={false}` disables the top input for fixed-rate display (e.g. "1 BTC"). Arrows badge uses `$accentYellowPrimary` background with `$black100` icon color (mode-invariant), centered via absolute overlay with `pointerEvents="none"`. Fiat field shows `$accentHighlightSecondary` border when focused. Frame styled with `$backgroundSecondary`, radius `$5`, height `$11`.
  - `Tabs` — Horizontal tab bar (large/small sizes). Generic over `T` — accepts `tabs: ReadonlyArray<TabItem<T>>` (each with `label` + `value`), `activeTab: T`, `onTabPress: (value: T) => void`. Animated underline slides between tabs via react-native-reanimated (`useSharedValue`/`withTiming`). Wrapped in RN `ScrollView` (horizontal) for overflow. Tab labels use heading font (`$heading`) with bold/regular weight toggle. Measures tab positions via `onLayout` and tracks ScrollView width for auto-scroll on press.
  - `InputHint` — Hint text row displayed below inputs. 2 variants (`InputHintVariant`): default (`$foregroundSecondary`) and error (`$redForeground`). Optional `showIcon` prop (default true) renders a leading icon (defaults to `FlagReport`, customizable via `icon` prop as `React.ComponentType<IconProps>`). Icon color resolved via `useTheme()`. Typography: body Micro (size `$1`, weight 500).
  - `IconTag` — Small icon-only tag with asymmetric border radii (right-leaning shape). 3 typed variants (`IconTagVariant`): bitcoin, product, service. Each variant binds a fixed icon (`CurrencyBitcoinCircle`, `BoxProduct`, `Tools`) and colors — bitcoin uses `$accentYellowSecondary` bg with `$accentHighlightSecondary` icon; product/service use `$backgroundHighlight` bg with `$foregroundPrimary` icon. Icon color resolved via `useTheme()`.
  - `TextTag` — Small status tag with asymmetric border radii. 10 typed variants (`TextTagVariant`): offer, request, approved, waiting, new, paused, set, accepted, waitingForConfirmation, outdated. Each variant binds background color, text color, and border-radius shape (left-leaning for offer/request, right-leaning for all others). Uses coordinated `styled()` variants on both frame (`TextTagFrame`) and label (`TextTagLabel`). Typography: body font, weight 500, size `$1` (12px).
  - `Picker` — Dropdown picker with tap-to-toggle item list. Generic over `T extends string` — accepts `items: ReadonlyArray<PickerItem<T>>` (each with `label` + `value` + optional `icon`), `value: T`, `onValueChange: (value: T) => void`. Trigger: `$backgroundSecondary` bg, height `$11`, radius `$5`, gap `$3`. Shows selected item's icon (if any, 24px) + label (`$accentHighlightPrimary`, body Paragraph size `$4`, weight 500) + `ChevronDown`. Dropdown content: `$backgroundSecondary` bg, radius `$5`, padding `$4`, gap `$4`. Items stretch full width with padding (`$2` vertical, `$3` horizontal), radius `$2.5`, press highlight `$backgroundTertiary`. Item text only (no icons in dropdown): body Description (size `$2`, weight 500, `$foregroundPrimary`). Animated open/close via `react-native-reanimated` — height-based clip reveal using `position: 'absolute'` inner content for correct measurement, animated `height` on outer clip container. Uses `scheduleOnRN` from `react-native-worklets` (not deprecated `runOnJS`) for animation completion callbacks. Custom implementation using `styled()` primitives with `useState` for open/close — Tamagui Select's floating content does not work reliably on React Native.
  - `RowButton` — Full-width row button with leading icon and label. 2 states: default (`$backgroundSecondary` bg, `$accentHighlightPrimary` text) and selected (`$accentYellowSecondary` bg, `$accentHighlightSecondary` text). Uses coordinated `styled()` variants on frame (`RowButtonFrame`) and label (`RowButtonLabel`). Optional `icon` prop (`React.ComponentType<IconProps>`). Height `$11` (56px), padding `$5`, radius `$5`, gap `$3`. Typography: body Paragraph (size `$4`, weight 500).
  - `RowCheckbox` — Full-width row checkbox with label and optional description. 2 states: default (`$backgroundSecondary` bg, `$foregroundPrimary` text, `SquareOutline` icon) and checked (`$accentYellowSecondary` bg, `$accentHighlightSecondary` text, `CheckboxFilled` icon). Uses Tamagui `Checkbox` (unstyled) as the base pressable checkbox element; the entire row is also pressable. Coordinated `styled()` variants on frame, label, and description. Controlled via `checked`/`onCheckedChange` props. Padding `$5`, radius `$5`, gap `$3`. Label: body Paragraph (size `$4`, weight 500). Description: body Micro (size `$1`, weight 500).
  - `RowRadiobutton` — Full-width row radio button with label and optional description. 2 states: default (`$backgroundSecondary` bg, `$foregroundPrimary` text, `RadiobuttonCircleEmpty` icon) and selected (`$accentYellowSecondary` bg, `$accentHighlightPrimary` text, `RadiobuttonCircleFilled` icon). Uses Tamagui `RadioGroup.Item` (unstyled) as the base pressable element — must be placed inside a `RadioGroup` parent. Controlled via `value`/`selected` props; parent `RadioGroup` manages `value`/`onValueChange`. Coordinated `styled()` label and description with `selected` variant. Padding `$5`, radius `$5`, gap `$3`. Description indented `$8` (32px) to align with label text past the icon. Label: body Paragraph (size `$4`, weight 500). Description: body Description (size `$2`, weight 500).
  - `SelectableItem` — Full-width selectable list item with label, optional note, and radio icon. 2 states: default (`$foregroundSecondary` text, no icon) and selected (`$accentHighlightSecondary` text, weight 600, `RadiobuttonCircleFilled` icon). Bottom border in `$backgroundSecondary`. Opacity press feedback (`pressStyle`). Controlled via `selected`/`onPress` props — parent manages single-selection (radio) behavior. Note text: body Micro (size `$1`, weight 500, `$foregroundTertiary`). Label: body Paragraph (size `$4`).
  - `SearchBar` — Pill-shaped search input using Tamagui `Input` (with `unstyled: true`). Takes jotai `valueAtom` (`WritableAtom<string, [SetStateAction<string>], void>`) for text state. Shows `SearchMagnifyGlass` icon on the left, `XmarkCancelClose` clear button on the right when text is non-empty (clears atom on press). Frame is `styled(XStack)` with `$backgroundSecondary`, pill radius (`$11`), height `$10` (48px), `alignSelf: 'stretch'` (fills parent width). Inner input is `styled(Input, { unstyled: true })` with Tamagui token references (`$body` font, `$4` size, `$foregroundPrimary` color). Theme colors resolved via `useTheme()` for `placeholderTextColor` and `selectionColor` (`$accentYellowPrimary`). Placeholder defaults to "Search" with `$foregroundPrimary` color.
  - `TextField` — Text input field using Tamagui `Input` (with `unstyled: true`). Takes jotai `valueAtom` for text state. Supports 5 right-side element modes: none (plain), `showClear` (X icon clears text when non-empty), `onCheckmarkPress` (yellow checkmark button + `$accentHighlightSecondary` border on focus), `buttonLabel`+`onButtonPress` (small secondary `Button` with compact frame padding), and `icon`+`onIconPress` (custom icon with press action). Frame is `styled(XStack)` with `$backgroundSecondary`, radius `$5` (16px), height `$11` (56px) for standard layout or auto for compact (text button). Uses 1px transparent border (avoids layout shift when focus border appears). Icon color for clear/custom uses `$foregroundPrimary`; checkmark icon uses `$accentHighlightPrimary`. Focus tracking via `useState` for border highlighting.
  - `Selector` — Full-width row selector with two variants: `action` (shows `ChevronRight`, fires `onPress`, opacity press feedback) and `switch` (shows `Switch` toggle, uses jotai `valueAtom`, no press feedback — behaves like `TouchableWithoutFeedback`). Optional leading `icon` prop (`React.ComponentType<IconProps>`). Uses separate styled frames per variant (`SelectorActionFrame` with `pressStyle`, `SelectorSwitchFrame` without). Switch variant toggles on row press and via native Switch. Height `$11` (56px), padding `$5`, radius `$5`, gap `$3`. Label: body Paragraph (size `$4`, weight 500, `$foregroundPrimary`). Chevron color: `$foregroundSecondary`. Icon color: `$foregroundPrimary`.
  - `SegmentedPicker` — Generic segmented toggle picker. Generic over `T` — accepts `tabs: SegmentedPickerTab<T>[]` (each with `label` + `value`), `activeTab: T`, `onTabPress: (value: T) => void`. Supports 2+ segments. No press feedback (no `pressStyle`) — behaves like `TouchableWithoutFeedback`. Uses coordinated `styled()` variants on frame (`SegmentFrame`) and label (`SegmentLabel`) with `selected` boolean variant. Outer frame has `borderRadius: '$5'` with `overflow: 'hidden'`; inner segments have no border radius. 1px gap (`$0.5`) between segments. Selected: `$accentYellowSecondary` bg, `$accentHighlightPrimary` text, weight 600. Unselected: `$backgroundSecondary` bg, `$foregroundPrimary` text, weight 500. Height `$11` (56px). Typography: body Paragraph (size `$4`).
  - `Menu` — Container that groups `MenuItem` children with auto-inserted `MenuDivider` separators. Provides `$backgroundSecondary` bg, radius `$5`, padding `$4`/`$5`, gap `$5`. Uses `MenuContext` (React context) so nested `MenuItem` components automatically strip their own bg/radius/padding (`contained` variant). Dividers use `$backgroundTertiary`, 1px height. Accepts `children: React.ReactNode`.
  - `MenuItem` — Full-width menu row with optional leading icon, label, optional note subtitle, optional trailing tag (e.g. `TextTag`), and optional trailing chevron. 2 variants (`MenuItemVariant`): default and danger. Supports `contained` boolean variant (auto-applied when inside `Menu`) — strips bg/radius/padding so the `Menu` container provides them. Container styled with `$backgroundSecondary` bg, radius `$5`, padding `$4`/`$5`. Opacity press feedback (`pressStyle`). Default: icon `$foregroundPrimary`, chevron `$foregroundSecondary`, label `$foregroundPrimary`, note `$foregroundSecondary`. Danger: all icon/label/note/chevron colors use `$redForeground`. Label: body Paragraph (size `$4`, weight 500). Note: body Micro (size `$1`, weight 500).
  - `Toast` — Animated toast/snackbar notification. Takes a jotai `messageAtom` (`WritableAtom<string | null, ...>`); when set to a non-null string, fades in at the top of the screen, displays for 3 seconds, then auto-dismisses with fade-out. Clears the atom immediately after capturing the message so re-triggers work (even same text). Uses `react-native-reanimated` (`useSharedValue`/`withTiming`) for opacity animation. Pill styled with `$foregroundPrimary` background and `$backgroundPrimary` text (inverted colors for contrast in both themes). Uses plain RN `Text` with `bodyFont` config values (size 14px, weight 600). Optional `topOffset` prop for safe-area clearance.

Design tokens sourced from Figma file `P7IaNcwu4qoS9uTL7ECiWL` (Vexl redesign DEV, node 593:39715):

- **Colors**: 24 semantic theme tokens (light/dark) across accents, background, foreground, vibrant groups.
- **Spacing**: 14-step scale (0px–80px).
- **Corner radius**: 13-step scale (0px–40px).
- **Typography**: TT Satoshi (body, weights 500/600) and Monument Extended (heading, weights 400/700).

Component patterns:

- Purely visual components use `styled()` with a `name` property matching the export name.
- Components needing coordinated styling (e.g. frame + label sharing a variant) use a functional component that composes internal styled primitives and forwards variant props to each.
- Use theme tokens (`$foregroundPrimary`) over raw color tokens (`$black100`) unless the design intentionally avoids theme-switching (e.g. CardButton filled without contrast).
- When a variant must ignore the surrounding theme, wrap it in `<Theme name="light">` (see NavButton normal variant).
- For animated components, use `react-native-reanimated` (shared values, `useAnimatedStyle`, `withTiming`/`withSequence`/`withRepeat`). When Animated.View needs token-derived pixel values, resolve via `getTokens()` from tamagui (e.g. `getTokens().size.$3.val`). Wrap token `.val` lookups in `useMemo` to avoid recalculating on every render. Do NOT wrap `.val` in `Number()` or `String()` — token values are already the correct type. For scheduling JS callbacks from worklet animations, use `scheduleOnRN` from `react-native-worklets` (NOT `runOnJS` — it is deprecated). Note: `scheduleOnRN(fn, ...args)` calls directly (returns void), unlike the old `runOnJS(fn)(...args)` curried pattern.
- For image filters (grayscale, color transforms), use `react-native-svg` SVG filters (`Filter`, `FeColorMatrix`, etc.) instead of third-party native filter libraries — they work with Fabric/New Architecture out of the box. Render images via SVG `<Image>` with `preserveAspectRatio="xMidYMid slice"` for cover-like behavior.
- For icon props, pass the component type (`React.ComponentType<IconProps>`) not a rendered element. The consuming component destructures with a capital alias (`icon: Icon`) and renders `<Icon color={...} size={...} />` directly. Never use `React.cloneElement` for color injection.

Gotchas:

- This is a native-only package. Do NOT add `@tamagui/react-native-media-driver` or media queries.
- Do NOT copy from the existing mobile app tamagui config — the new design system is separate.
- Font family names (`TTSatoshi`, `MonumentExtended`) are placeholders — actual font files need loading via expo-font in consuming apps.
- Always re-export the tamagui config from `src/config/tamagui.config.ts` so consuming apps can reference it in their Babel plugin.
- Use `React.JSX.Element` return type (not bare `JSX.Element`) — the generic-esm tsconfig doesn't provide the global JSX namespace.
- After changes, run `yarn workspace @vexl-next/ui typecheck` to verify.
